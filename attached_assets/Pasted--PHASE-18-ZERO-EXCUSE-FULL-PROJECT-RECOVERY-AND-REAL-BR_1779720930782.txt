3. الاختبار الحاسم: Real Browser Admin Login
==================================================

أنشئ اختبار Playwright حقيقي، مثلاً:

tests/e2e/admin-real-browser-login.spec.ts

أو أي مسار مناسب.

الاختبار يجب أن يفتح المتصفح فعلياً ويختبر DOM الحقيقي:

1. افتح /admin?reset=1
2. انتظر الرجوع إلى /admin
3. تحقق أن صفحة login ظاهرة
4. املأ email من ADMIN_E2E_EMAIL
5. املأ password من ADMIN_E2E_PASSWORD
6. اضغط زر "تسجيل الدخول"
7. انتظر نتيجة خلال 10 ثوانٍ

النتائج المقبولة فقط:

A. دخول Dashboard:
- URL يحتوي /admin/dashboard أو محتوى dashboard ظاهر.
- Admin shell ظاهر.
- شعار مواعيدك ظاهر.
- sidebar أو dashboard cards ظاهرة.
- الحكم المرحلي: browser admin login passed.

B. Access Denied:
- role ظاهر في الواجهة أو debug box.
- session موجودة.
- الحكم: Needs Supabase Role Fix.

C. Invalid Credentials:
- رسالة بيانات الدخول غير صحيحة تظهر.
- الحكم: Needs Owner Password Reset.

D. Timeout / Loading / No UI Change:
- هذا فشل كود.
- أصلح AdminLayout/Login.
- أعد الاختبار حتى لا يحدث.

ممنوع اعتبار REST HTTP 200 وحده نجاحاً.
النجاح الحقيقي هو أن واجهة المتصفح تدخل Dashboard أو تعطي خطأ واضحاً.

==================================================
4. إذا REST Auth ينجح لكن Browser Login يفشل
==================================================

هذا هو الاحتمال الأهم. إذا حدث، شخّص وأصلح:

1. هل frontend يستخدم نفس VITE_SUPABASE_URL الذي استخدمه السكربت؟
2. هل VITE_SUPABASE_ANON_KEY في المتصفح صحيح؟
3. هل import.meta.env يقرأ القيم فعلاً؟
4. هل supabase client في المتصفح null؟
5. هل form submit يعمل؟
6. هل button type="submit"؟
7. هل onSubmit يُستدعى؟
8. هل setSubmitting(false) يحدث في finally؟
9. هل session تُحفظ في localStorage بعد signIn؟
10. هل getUser يرجع user؟
11. هل role parsing صحيح؟
12. هل navigate("/admin/dashboard") يعمل؟
13. هل router يمنع التحويل؟
14. هل AdminLayout يعيد phase إلى login بعد نجاح الدخول؟
15. هل onAuthStateChange يسبب race condition؟
16. هل INITIAL_SESSION يسبب loop؟
17. هل هناك overlay أو pointer-events يمنع الضغط؟
18. هل هناك preventDefault ناقص؟
19. هل الزر disabled بسبب state قديم؟
20. هل /admin/dashboard route موجود ويُعرض؟

أصلح السبب الفعلي داخل الكود ثم أعد Playwright test.

==================================================
5. إصلاح Admin Auth Flow إلزامي
==================================================

يجب أن يكون AdminLayout أو AdminLogin كالتالي:

Phases واضحة:
- checking
- login
- submitting
- ready
- access_denied
- auth_error

قواعد:
- checking لا يتجاوز 3 ثوانٍ.
- submitting لا يتجاوز 8 ثوانٍ.
- لا loading أبدي.
- لا زر معلق.
- لا شاشة بيضاء.
- كل Promise له timeout.
- كل مسار له finally.
- reset يعمل حتى لو Supabase signOut فشل.

زر الدخول:
- داخل form
- type="submit"
- onSubmit حقيقي
- native HTML button أو button مضمون

زر مسح الجلسة:
- type="button"
- يمسح localStorage keys التي تحتوي supabase / sb- / auth
- sessionStorage.clear()
- يرجع phase إلى login
- لا يعلق

/admin?reset=1:
- يمسح الجلسة تلقائياً بدون زر
- يرجع إلى /admin
