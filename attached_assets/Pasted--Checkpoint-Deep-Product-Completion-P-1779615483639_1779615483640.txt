==================================================
1. الهدف التنفيذي
==================================================

حوّل التطبيق إلى Web/PWA أكثر جاهزية واستقراراً من ناحية تجربة المستخدم، الخطأ، الوصول، والجوال.

المطلوب تنفيذ أو التحقق من التالي:

1. PWA basics:
   - manifest صحيح.
   - name.
   - short_name.
   - theme_color.
   - background_color.
   - icons أو placeholders آمنة.
   - display مناسب.
   - start_url.
   - lang أو dir عند الإمكان.
   - service worker أو توثيق سبب عدم توفره.
   - offline fallback بسيط إن أمكن.

2. Error Handling:
   - Error Boundary عام.
   - لا شاشة بيضاء عند الخطأ.
   - رسالة خطأ عربية واضحة.
   - زر رجوع للرئيسية.
   - صفحة 404 عربية للمسارات غير الموجودة.

3. Loading / Empty / Error States:
   - حالات تحميل واضحة.
   - حالات فراغ واضحة.
   - حالات خطأ واضحة.
   - لا تظهر صفحات فارغة بلا تفسير.

4. Accessibility:
   - aria-label للأزرار الأيقونية.
   - focus states.
   - تباين ألوان جيد.
   - أحجام لمس مناسبة للجوال.
   - عدم الاعتماد على اللون وحده.
   - دعم RTL في الحقول والقوائم والجداول.
   - النصوص العربية واضحة وغير مقطوعة.

5. Mobile UX:
   - Bottom navigation لا يغطي المحتوى.
   - padding سفلي كافٍ.
   - لا يوجد overflow أفقي.
   - الجداول أو البطاقات قابلة للاستخدام على الجوال.
   - العناصر اللمسية مناسبة.

6. توثيق ما تم في QA_REPORT.

==================================================
5. قاعدة التنفيذ الآمن
==================================================

نفذ بأقل تغيير آمن.

القواعد:
- أي تحسين يجب ألا يكسر الوظائف المعتمدة.
- Error Boundary يجب أن يظهر رسالة عربية بدل الشاشة البيضاء.
- صفحة 404 يجب ألا تكسر المسارات الموجودة.
- manifest يجب أن يكون آمناً ولا يحتوي روابط خارجية غير لازمة.
- service worker إن أضيف يجب ألا يكسر API أو تحديثات التطبيق.
- offline fallback إن أضيف يجب أن يكون بسيطاً وصادقاً.
- لا تجعل الزخرفة تقلل التباين أو القراءة.
- لا تكسر RTL.
- لا تكسر Mobile-first.

==================================================
7. المتطلبات التفصيلية
==================================================

أ. PWA:
- تحقق أو أضف manifest.
- أضف icons placeholders آمنة إن لم توجد.
- استخدم اسم التطبيق:
  مواعيدك
- استخدم short_name:
  مواعيدك
- theme_color مناسب للثيم الحالي.
- background_color مناسب.
- display: standalone أو ما يناسب.
- start_url صحيح.
- وثّق service worker:
  - مفعّل ويعمل.
  أو
  - غير متوفر/مؤجل بسبب قيود البيئة.

ب. Error Boundary:
- مكوّن عام يلتقط أخطاء الواجهة.
- رسالة عربية واضحة.
- زر رجوع للرئيسية.
- لا يعرض تفاصيل تقنية للمستخدم العادي.
- يمكن توثيق الخطأ في console للتطوير فقط.

ج. 404:
- صفحة عربية.
- نص واضح أن الصفحة غير موجودة.
- زر العودة للرئيسية.
- تصميم مطابق للثيم.
- لا شاشة بيضاء.

د. Loading / Empty / Error:
- مكونات أو أنماط موحدة.
- لا صفحات فارغة بلا رسالة.
- حالات خطأ للبيانات عند فشل التحميل.
- حالات فراغ للمراكز، الإشعارات، الأخبار، الوظائف، المهام، الشكاوى حسب الحاجة.

هـ. Accessibility:
- aria-label للأزرار الأيقونية:
  - الإشعارات.
  - المشاركة.
  - الحذف.
  - التعديل.
  - القائمة.
  - إخفاء/إظهار.
- focus states واضحة.
- التباين مقبول.
- touch targets مناسبة.
- الحقول لها labels واضحة.
- عدم الاعتماد على اللون وحده في الحالة.

و. Mobile:
- فحص الجوال.
- لا overflow أفقي.
- bottom nav لا يغطي المحتوى.
- الجداول قابلة للتمرير أو التحويل إلى cards.
- padding سفلي كافٍ.
