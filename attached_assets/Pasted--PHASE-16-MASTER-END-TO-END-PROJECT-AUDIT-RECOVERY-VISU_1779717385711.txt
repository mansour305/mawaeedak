==================================================
B. ADMIN ACCESS RECOVERY — BLOCKER
==================================================

هذه أولوية قصوى.

المشكلة العملية:
المستخدم يقول إن /admin لا يدخل ويعلق على “جارٍ التحقق”.

المطلوب:
إصلاح /admin بحيث يستحيل أن يعلق إلى الأبد.

القاعدة:
شاشة “جارٍ التحقق” لا يجوز أن تستمر أكثر من 3 ثوانٍ.

بعد 3 ثوانٍ يجب أن يحدث واحد من التالي فقط:
1. Dashboard إذا session + role صحيحة.
2. Login Form إذا لا توجد session.
3. Access Denied إذا role غير صحيحة.
4. Auth Error مع زر مسح الجلسة.

ممنوع أي حالة خامسة.

نفّذ التالي:

1. راجع AdminLayout.
2. راجع auth.ts.
3. راجع hasAdminAccess.
4. راجع getAuthSession.
5. راجع authSignIn.
6. راجع onAuthStateChange.
7. راجع redirect behavior.
8. راجع direct routes:
   - /admin
   - /admin/dashboard
   - /admin/automation
   - /admin/visual-guide
   - /admin/data-layer

استخدم state machine صريح:

type AdminAuthPhase =
  | "checking"
  | "login"
  | "ready"
  | "access_denied"
  | "auth_error";

كل phase يجب أن يرسم UI واضحاً.

أضف timeout حقيقي 3 ثوانٍ:
- إذا بقي phase = checking بعد 3 ثوانٍ:
  - setPhase("login")
  - اعرض رسالة:
    "انتهت مهلة التحقق، سجّل الدخول مرة أخرى"

أضف زر:
"مسح الجلسة والبدء من جديد"

ويفعل:
- supabase.auth.signOut({ scope: "local" })
- حذف localStorage keys التي تحتوي:
  - supabase
  - sb-
  - auth
- sessionStorage.clear()
- setPhase("login")
- redirect إلى /admin

تحقق من role من:
1. user.user_metadata.role
2. user.app_metadata.role

اقبل فقط:
- super_admin
- admin

إذا role غير موجود:
اعرض Access Denied واضحاً مع SQL Fix:

UPDATE auth.users
SET 
  raw_user_meta_data = COALESCE(raw_user_meta_data, '{}'::jsonb) || '{"role": "super_admin"}'::jsonb,
  raw_app_meta_data  = COALESCE(raw_app_meta_data, '{}'::jsonb)  || '{"role": "super_admin"}'::jsonb
WHERE email = 'hrq@hotmail.com';

مهم:
حتى لو role ناقص، ممنوع أن يعلق /admin.
يجب أن يعرض Access Denied أو Login.
