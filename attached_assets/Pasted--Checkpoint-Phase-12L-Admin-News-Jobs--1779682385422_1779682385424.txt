==================================================
5. متطلبات التنفيذ
==================================================

أ. Supabase adapter:
أضف في supabaseData.ts دوال آمنة مثل:
- createNewsInSupabase
- updateNewsInSupabase
- deleteNewsInSupabase
- createJobInSupabase
- updateJobInSupabase
- deleteJobInSupabase

ب. Data Gateway:
أضف في dataGateway.ts:
- gwCreateNews
- gwUpdateNews
- gwDeleteNews
- gwCreateJob
- gwUpdateJob
- gwDeleteJob

ج. السلوك حسب mode:
- api:
  API فقط.
- supabase_shadow:
  API فقط.
- supabase:
  Supabase فقط لنطاق news/jobs.
  لا fallback صامت للكتابة إلى API.

د. Query invalidation:
بعد أي عملية:
- حدّث gw news/jobs query keys.
- حدّث Orval query keys للتوافق إن بقيت بعض الصفحات API.
- لا تترك stale cache.

هـ. IDs:
- إذا واجهة admin تستخدم legacy integer id، استخدم legacy_id في Supabase.
- إذا احتجت uuid، وثّق ذلك ولا تكسر الواجهة.
- الأفضل لهذه المرحلة: الحفاظ على legacy_id كـ id المعروض للواجهة.
