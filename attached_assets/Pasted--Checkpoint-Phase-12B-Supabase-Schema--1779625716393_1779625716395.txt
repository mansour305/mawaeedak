4. القرار المعماري المطلوب
==================================================

اعتمد القرار التالي ما لم يظهر مانع تقني واضح:

1. Supabase يستخدم uuid كـ primary id مستقبلياً.
2. نحافظ على id القديم من PostgreSQL داخل عمود:
   legacy_id integer
3. الجداول الشخصية تأخذ:
   user_id uuid references auth.users(id)
4. بيانات المستخدم القديمة عند migration ستُربط مؤقتاً بـ user_id للمالك:
   hrq@hotmail.com
   أو بخطة owner assignment موثقة.
5. admin-managed tables لا تحتاج user_id، لكن تحتاج legacy_id عند النقل.
6. لا يتم تغيير Drizzle schema الحالي في هذه المرحلة إلا إذا كان توثيقياً أو غير مؤثر.
7. Data Layer الحالي يبقى كما هو.
