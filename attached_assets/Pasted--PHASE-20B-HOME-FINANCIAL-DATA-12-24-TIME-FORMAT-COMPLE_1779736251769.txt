==================================================
1. إصلاح القسم المالي في HomePage
==================================================

المطلوب:
بطاقات الرواتب والدعوم يجب أن تظهر في الرئيسية مثل المرجع.

المشكلة الحالية:
في mode=supabase، HomePage لا يعرض financial cards لأن Supabase لا تُرجع بيانات للمستخدم الحالي أو gateway يُعيد null.

الإصلاح المطلوب:
افحص gwGetFinancialCountdown / useGatewayFinancialCountdown / HomePage.

يجب أن تعمل HomePage كالتالي:

1. في mode=supabase:
   - اقرأ financial_events من Supabase للمستخدم الحالي.
   - إذا لا توجد session أو لا توجد صفوف:
     استخدم fallback آمن إلى API للعرض فقط، أو استخدم public financial countdown endpoint المصنف API intentionally.
   - لا تترك القسم فارغاً.

2. يجب أن تظهر على الأقل:
   - راتب مايو / الراتب الشهري
   - حساب المواطن أو الدعم السكني
   - التأهيل الشامل أو الضمان
   - بطاقة رابعة إن وجدت

3. إذا لا توجد بيانات حقيقية:
   - اعرض رسالة صغيرة:
     "لا توجد مواعيد مالية مخصصة"
   - لكن لا تُخفي القسم بالكامل.
   - ولا تجعل الشاشة تختلف عن المرجع بشكل واضح.

الأفضل:
استخدم endpoint:
GET /api/financial-events/countdown
كـ fallback للعرض في HomePage لأنه server-computed وموجود ويعيد 200.

المطلوب بصرياً:
- بطاقة راتب رئيسية بعد شريط الصلاة.
- عداد يوم/ساعة/دقيقة/ثانية.
- بطاقات دعم عمودية.
- زر "عرض التفاصيل".
- لا empty state كبير في الرئيسية.
