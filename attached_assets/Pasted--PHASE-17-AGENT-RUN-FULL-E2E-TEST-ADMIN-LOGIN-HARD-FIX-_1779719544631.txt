==================================================
3. إذا لم تتحرك الأزرار
==================================================

إذا ضغطت الأزرار ولم تتغير العدادات أو لم تظهر نتيجة، أصلح فوراً:

- أزل أي overlay فوق الفورم.
- أزل pointer-events: none من أي parent.
- لا تستخدم زر custom معطل.
- استبدل زر الدخول بزر HTML أصلي.
- استبدل زر reset بزر HTML أصلي.
- اجعل الفورم native:
  <form onSubmit={handleLoginSubmit}>
- اجعل زر الدخول:
  <button type="submit">
- اجعل زر مسح الجلسة:
  <button type="button" onClick={handleHardResetAuth}>
- ضع console.info في أول سطر من كل handler.
- ضع onClick احتياطي على زر الدخول يستدعي نفس submit handler عند الحاجة.
- لا تترك submit يعتمد على مكوّن Button مخصص.

المطلوب:
حتى لو كان CSS أو component هو سبب المشكلة، يجب أن يعمل الضغط فعلياً.

==================================================
4. أصلح Login بطريقة لا تعلق
==================================================

handleLoginSubmit يجب أن يكون بسيطاً وحاسماً:

- event.preventDefault()
- increment submitFiredCount فوراً
- validate email/password
- setSubmitting(true)
- signInWithPassword عبر Supabase مباشرة
- Promise.race timeout 8000ms
- getUser timeout 5000ms
- read role من user_metadata ثم app_metadata
- role صحيح: phase ready + navigate /admin/dashboard
- role ناقص: access_denied واضح
- password خطأ: login error واضح
- network/timeout: auth_error واضح
- finally دائماً setSubmitting(false)

ممنوع أي loading أبدي.

==================================================
5. أصلح Reset بطريقة لا تعلق
==================================================

handleHardResetAuth يجب أن يكون فورياً:

- increment resetFiredCount فوراً
- console.info("[AdminLogin] reset fired")
- setSubmitting(false)
- clear errors

- حذف localStorage keys التي تحتوي:
  supabase

- setPhase("login")
- clear password
- history.replaceState إلى /admin

حتى لو Supabase signOut فشل، يجب أن يكمل المسح المحلي ولا يعلق.
