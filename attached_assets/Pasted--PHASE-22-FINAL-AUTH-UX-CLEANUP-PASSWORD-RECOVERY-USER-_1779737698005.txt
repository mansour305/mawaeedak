==================================================
2. إضافة نسيت كلمة المرور
==================================================

أضف Flow كامل لاستعادة كلمة المرور.

المطلوب:

1. في شاشة الدخول أضف رابط:
"نسيت كلمة المرور؟"

2. عند الضغط:
يعرض نموذج بسيط:
- حقل البريد الإلكتروني
- زر "إرسال رابط الاستعادة"
- زر رجوع لتسجيل الدخول

3. عند إدخال البريد:
استخدم Supabase password recovery:
resetPasswordForEmail

4. الرسالة بعد الإرسال:
"إذا كان البريد مسجلاً، سيصلك رابط استعادة كلمة المرور."

مهم أمنياً:
لا تكشف هل البريد موجود أو غير موجود.

5. أضف صفحة أو route لتحديث كلمة المرور بعد الرجوع من رابط Supabase، مثل:
- /reset-password
أو route مناسب.

6. صفحة تحديث كلمة المرور يجب أن تحتوي:
- كلمة المرور الجديدة
- تأكيد كلمة المرور
- زر "تحديث كلمة المرور"
- تحقق من تطابق الكلمتين
- تحقق من الحد الأدنى للطول
- رسالة نجاح
- تحويل بعدها إلى /admin أو /account حسب السياق

7. لا تعرض access_token أو refresh_token في الشاشة.
8. لا تطبع tokens في console.
9. لا تحفظ كلمة المرور في localStorage.

==================================================
3. إضافة تسجيل عضو جديد
==================================================

أضف Flow تسجيل مستخدم جديد.

المهم:
"تسجيل عضو جديد" لا يعني مالك أو Admin.
أي مستخدم جديد يجب أن يكون role = user فقط، ولا يدخل لوحة المالك.

المطلوب:

1. في شاشة الدخول أضف رابط:
"تسجيل عضو جديد"

2. عند الضغط:
يعرض نموذج تسجيل:
- الاسم
- البريد الإلكتروني
- كلمة المرور
- تأكيد كلمة المرور
- الموافقة على الشروط وسياسة الخصوصية

3. عند التسجيل:
استخدم Supabase signUp.

4. user_metadata للمستخدم الجديد:
- role: "user"
- name: الاسم المدخل

5. ممنوع:
- لا تضبط role=admin
- لا تضبط role=super_admin
- لا تسمح للمستخدم الجديد بدخول /admin/dashboard

6. بعد التسجيل:
إذا Supabase يحتاج تأكيد بريد:
اعرض:
"تم إنشاء الحساب. تحقق من بريدك لتفعيل الحساب."

إذا لا يحتاج تأكيد:
اعرض:
"تم إنشاء الحساب بنجاح."
ثم وجّه المستخدم إلى صفحة مناسبة مثل /account أو /.

7. إذا حاول المستخدم الجديد فتح /admin:
يظهر له Access Denied واضح:
"هذا الحساب لا يملك صلاحية لوحة المالك."

==================================================
4. فصل دخول المالك عن تسجيل العضو
==================================================

يجب توضيح الفرق:

- تسجيل الدخول في /admin خاص بلوحة المالك.
- تسجيل العضو الجديد ينشئ حساب مستخدم عادي.
- المستخدم العادي لا يدخل لوحة المالك.
- super_admin فقط يدخل لوحة المالك.

في UI:
لا تستخدم مصطلحات تقنية.
استخدم لغة عربية واضحة.

مثال:
"لوحة المالك مخصصة لإدارة التطبيق. إذا كنت مستخدماً جديداً، يمكنك إنشاء حساب عادي."

5. حماية Admin
==================================================

بعد إضافة التسجيل، اختبر:

1. super_admin يدخل /admin/dashboard.
2. user عادي لا يدخل /admin/dashboard.
3. user عادي يرى Access Denied.
4. لا يمكن ترقية user إلى admin من الواجهة.
5. role parser لا يقبل إلا:
   - admin
   - super_admin
للوحة المالك.
